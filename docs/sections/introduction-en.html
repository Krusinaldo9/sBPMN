Introduction back to ToC

The sBPMN ontology provides a semantic representation of business processes modeled using the BPMN 2.0 standard. It enables the transformation of BPMN process models into expressive and interoperable knowledge graphs that can be used for semantic querying, explainability, and process monitoring.

The ontology is organized into two aligned modules:

  • sbpmnc defines the BPMN classes.
  • sbpmnp defines the properties used to assert process-specific facts.

This separation reflects the structure of the official BPMN 2.0 XML schema, where elements and attributes may share the same local name but differ in their syntactic role. By mirroring this distinction in OWL, sBPMN ensures semantic compatibility with existing BPMN tools while enabling ontology-based reasoning and validation.

The combined use of sbpmnc and sbpmnp allows for the representation of both the static structure of process models and the dynamic aspects of their execution. This enables integration with runtime logs and external data sources, as well as support for federated queries over hybrid knowledge graphs.
